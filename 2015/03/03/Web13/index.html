入门到放弃13

H5

  • HTML是超文本语言 不是一种编程语言 而是一种标记语言 他通过标签语言 来标记网页结构或者说是内容
  • h5是h4的增强版就是在原来的基础上面增加了一些新的功能

语法规范

  • DOCUTYPE 指令告诉浏览器怎么处理我们的HTML h4很是复杂有很多写法 比如strict严格模式 loose 过渡版本 H5 最简单也最简洁也是现在最常用的写法

W3C验证地址：http://validator.w3.org/ 很好的一个检测东西

语义化

  • 传统的做法是以类名或者ID来语义
  • 语义化的好处 有利于搜索引擎或辅助设备
    • 页眉
      • 1、可以是整个页面或者任意”section”的头部部分。
      • 2、没有个数限制。
      • 3、如果h1-h6能很好的工作，就不要用header
    • 页脚
      • 1、可以是整个页面或者任意”section”的底部部分。
      • 2、没有个数限制。
      • 3、一般包含联系信息，版权信息等。
    • 导航
      • 整个页面的主要导航部分
    • 主要区域
      • 一般放在主要内容区域
    • 块区域
      • 1、表示文档的节或段。
      • 2、article、nav、aside可以理解为特殊的section。
      • 3、可以用article、nav、aside的就不用section。
    • 文章
      • 1、自身独立的情况下，用article。
      • 2、是相关内容，用section
      • 3、没有语义，用div
    • 侧边栏
      • 1、aside在main标签之内，和section或article标签对应，表示这些主要内容的附属信息。
      • 2、如果是广告，其他日志链接或者其他分类导航也可以使用。
    • 独立的流内容
      • 1、标签规定独立的流内容（图像、图表、照片、代码等等）。
      • 2、元素的内容应该与主内容相关，但如果被删除，则不应对文档流产生影响。
    • 独立流内容的标题
      • 1、定义figure中图片等东西的标题。
      • 2、元素应该被置于 “figure” 元素内的第一个或最后一个子元素的位置。

兼容处理

  • 在不支持H5新标签的浏览器中 会将这些标签解析成行内元素 所以我们只要转化成块级元素即可使用
  • IE9以下(不包括IE9)不能正常解析这些新标签 但是可以识别通过 这个时候我们可以创建元素的方式document.createElemenet的方式是可以创建出来 必须写在前面
  • 可以使用一个html5hiv.min的插件就可以完美解决这些问题
  • IE有个bug如果页面有中文在浏览器中访问的时候会编码然后就会没有效果 这时候你可以通过文件夹位置去访问就可以出现效果
  • n可以判断一下ie几才执行这些JS文件 不然引入太多JS文件会导致页面加载缓慢 用户体验不好

微数据

  • 给标签添加一些数据 然后是给机器看的比较友好
  • autofucus=”” 页面进入 定位焦点
  • pattern 验证 可以写个正则 实际验证的正则可以去其他网站扒人家的 不用自己写
  • file 上传文件只能一张 multiple使用了这个属性可以选择多张
  • 我们用新增的属性 可以完成我们之前要用JS才能实现的效果
  • 自动完成输入 记录上一次的输入 actocomplete=”on”
  • 表单里面的事件….
    • oninput 监听文本输入状态
    • oninvalid 当验证不通过的时候触发

自定义属性

  • 必须是data-开头 比如 data-yor_shift
  • document.getElementbyid(id).dataset[‘yorS hift’] 获取自定义属性 第一个大写也可以获取到

JS分为三个部分

  • ECMAScript
    • 定义了js的语法 变量 语句
  • BOM 浏览器对象模型
    • 我们可以调用一些自带的对象 就可以调出浏览器功能
  • DOM 文档对象模型
    • 操作dom的前期 就必须获得 dom
    • H5在获取标签的基础上进行了扩展 提供了一些更加强大的方法
    • docuement.querySelector 这个里面包含 css的选择器 可以是id也可以是类 返回的是第一个元素
      • docuement.querySelectorall 获取所有的 返回的是一个集合
      • 1、Node.classList.add(“class”) 添加class。
      • 2、Node.classList.remove(“class”) 移除class。
      • 3、Node.classList.toggle(“class”) 切换class，有则移除，无则添加。
      • 4、Node.classList.contains(“class”) 检测是否存在class。

1、视频

  • 属性：

    • paused: 视频的播放暂停状态
    • duration:视频的总时长

    • currentTime:当前播放时间

      • 方法： play() 播放
        • pause()暂停

      • 事件：

        • oncanplay： 当视频加载完成之后
        • ontimeupdate: 当视频当前时间 改变之后触发

      • 全屏：

        • video.webkitRequsetFullScreen();

    2、拖拽：

    • （1、需要设置拖拽：draggable=”true”
    • （2、拖拽元素：
      - ondragstart: 拖拽开始
- ondragend:拖拽结束
- ondragleave: 当鼠标拖拽离开 被拖拽元素时
- ondrag:　当拖拽时，持续触发
    • （3、目标元素：
      • ondragenter:当被拖拽元素进入目标元素时
      • ondragleave:当鼠标进入目标元素时
      • ondragover: 当被拖拽元素在目标元素上时，持续触发
        –> e.preventDefault(); 阻止拖拽的默认行为；
      • ondrop:当在目标元素上松开鼠标时触发；

        3、地理定位

    • navigator.getCurrentPosition(successCallback, errorCallback, options)获取当前地理信息

      • （1、当成功获取地理信息后，会调用succssCallback，
        并返回一个包含位置信息的对象position。Coords(坐标)
        position.coords.latitude纬度
        position.coords.longitude经度
      • （2、当获取地理信息失败后，会调用errorCallback，并返回错误信息error
      • （3、可选参数 options 对象可以调整位置信息数据收集方式

4、web存储

  • window.sessionStorage 会话存储
    • 生命周期为关闭浏览器窗口
    • 在同一个窗口下数据可以共享
    • sessionStorage约5M
  • window.localStorage
    • 永久生效，除非手动删除
    • 可以多窗口共享
    • localStorage约20M
  • 方法详解
    • setItem(key, value) 设置存储内容
    • getItem(key) 读取存储内容
    • removeItem(key) 删除键值为key的存储内容
    • clear() 清空所有存储内容

5、网络状态

  • window. ononline用户网络连接时被调用
  • Window.onoffline用户网络断开时被调用
